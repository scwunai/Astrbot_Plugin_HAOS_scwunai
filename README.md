# Astrbot_Plugin_HAOS_scwunai

> 智能家居助手：天气查询、传感器监控、设备控制、智能告警

Astrbot 的 HomeAssistant 智能家居集成插件，支持天气查询、传感器监控、设备控制，并通过 LLM 实现自然语言交互与智能回复。

## 功能特性

- 🌤️ **天气查询**：支持全国城市天气查询，包含当前天气、未来预报、分时天气
- 📊 **传感器监控**：实时监控 HomeAssistant 传感器状态，支持阈值告警
- 💡 **设备控制**：支持开关设备、空调控制（模式、温度、风速）
- 🔔 **智能告警**：传感器数据异常时自动推送告警消息
- 🤖 **自然语言**：支持多意图连续指令，LLM 智能生成自然回复
- 👤 **人格集成**：支持 AstrBot 人格系统，让回复更具个性
- ⏰ **定时推送**：每日定时推送天气信息，包含生活指数
- 🔐 **权限管理**：支持管理员用户/群组权限控制
- ⚡ **并行处理**：多意图并行请求，提高响应速度

## 安装

1. 将插件目录放置于 Astrbot 的 `plugins/` 目录下
2. 重启 Astrbot 或通过管理面板加载插件
3. 在插件配置页面填写 HomeAssistant 连接信息

## 配置说明

### 必填配置

| 配置项 | 说明 |
|-------|------|
| `home_assistant_url` | Home Assistant 地址，如 `http://homeassistant.local:8123` |
| `ha_token` | Home Assistant 长效访问令牌 |

### 传感器配置

在 `sensors` 列表中添加传感器：

```json
{
  "name": "客厅温度",
  "entity_id": "sensor.living_room_temperature",
  "sensor_type": "temperature",
  "unit": "°C",
  "low_threshold": 10,
  "high_threshold": 30,
  "enabled": true
}
```

### 设备配置

在 `switches` 列表中添加设备：

```json
{
  "name": "客厅灯",
  "entity_id": "switch.living_room_light",
  "type": "switch"
}
```

空调设备配置：

```json
{
  "name": "客厅空调",
  "entity_id": "climate.living_room_ac",
  "type": "ac",
  "min_temp": 16,
  "max_temp": 30
}
```

### 权限配置

| 配置项 | 说明 |
|-------|------|
| `admin_users` | 管理员用户列表，格式：`平台:用户ID` 或 `用户ID` |
| `admin_groups` | 管理员群组列表，格式：`平台:群ID` 或 `群ID` |
| `public_commands` | 公开指令列表，无需权限即可使用 |

### 可选配置

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| `weather_push_time` | `07:00` | 每日天气推送时间 |
| `sensor_check_interval` | `300` | 传感器检查间隔（秒） |
| `enable_sensor_alert` | `true` | 是否启用传感器异常告警 |
| `enable_weather_push` | `true` | 是否启用每日天气推送 |
| `enable_persona` | `false` | 是否启用 AstrBot 人格集成 |
| `persona_name` | `""` | 指定使用的人格名称，留空使用默认人格 |

### 人格集成配置

启用人格集成后，LLM 回复将使用 AstrBot 中配置的人格风格，使回复更加自然、有个性。

配置步骤：
1. 在 AstrBot 管理面板 → 人格管理中创建或查看人格
2. 在插件配置中启用 `enable_persona`
3. （可选）在 `persona_name` 中指定要使用的人格名称，留空则使用默认人格

## 指令列表

### 天气相关

| 指令 | 说明 |
|-----|------|
| `/set_location <城市>` | 设置用户位置 |
| `/weather` | 获取当前天气 |
| `/subscribe_weather` | 订阅每日天气推送 |
| `/unsubscribe_weather` | 取消天气订阅 |

### 传感器相关

| 指令 | 说明 |
|-----|------|
| `/sensor` | 查询所有传感器状态 |
| `/list_sensors` | 列出已配置的传感器 |
| `/air_quality` | 查询空气质量 |

### 设备控制

| 指令 | 说明 |
|-----|------|
| `/device` | 查询设备状态 |
| `/list_devices` | 列出已配置的设备 |
| `/turn_on <设备名>` | 打开设备 |
| `/turn_off <设备名>` | 关闭设备 |
| `/ac` | 查询空调状态 |
| `/ac <命令>` | 空调控制（制冷/制热/除湿/送风/关闭） |
| `/ac_temp <温度>` | 设置空调温度 |

### 其他

| 指令 | 说明 |
|-----|------|
| `/haoshelp` | 显示帮助信息 |

## 自然语言交互

### 单指令示例

- 「今天天气怎么样」→ 查询天气
- 「一个小时后天气怎么样」→ 查询分时天气
- 「两小时后我要出门，天气如何」→ 查询分时天气
- 「我在北京」→ 设置位置
- 「室内温度多少」→ 查询温度
- 「湿度多少」→ 查询湿度
- 「空气质量怎么样」→ 查询空气质量
- 「打开客厅灯」→ 打开设备
- 「关闭空调」→ 关闭空调
- 「空调开制冷」→ 设置空调模式
- 「空调温度调到26度」→ 设置空调温度
- 「订阅天气」→ 订阅天气推送

### 连续指令示例

支持在一条消息中包含多个指令，系统会依次执行并智能合并回复：

- 「打开空调，调成除湿模式，温度开到20度」
- 「打开客厅灯和卧室灯」
- 「关掉服务器、NAS电源」

### 智能分析示例

当用户提出分析类问题时，系统会结合查询数据进行智能分析：

- 「今天天气怎么样，结合卧室环境分析一下适合做什么」
- 「室内温度和湿度正常吗？」

系统会将天气数据、传感器数据发送给 LLM 进行分析，生成自然、有价值的回复。

## 获取 HomeAssistant Token

1. 登录 HomeAssistant 后台
2. 点击左下角用户名
3. 进入「安全」页面
4. 在「长期访问令牌」处创建新令牌

## 获取传感器 Entity ID

1. 进入 HomeAssistant 后台
2. 设置 → 设备与服务 → 实体
3. 找到目标传感器，复制实体标识符（如 `sensor.temperature`）
